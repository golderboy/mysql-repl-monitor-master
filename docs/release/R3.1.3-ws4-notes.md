# R3.1.3-ws4

## Scope
P3-WS4 Reporting และ executive outputs

## Added
- `/reports` page for HTML and JSON listing of report exports
- queued report export flow via `/reports/export`
- worker support for `report_export` jobs
- report manifest storage in `monitor_report_exports`
- CSV and PDF export generation without querying raw evidence tables directly
- download route for successful exports: `/reports/<report_id>/download`

## Database
Apply these SQL files in order:
1. `app/sql/006_reports.sql`
2. `app/sql/006_reports_grants.sql`

## Validation
```bash
python scripts/verify_patch.py
python -m unittest discover -s tests -p 'test_*.py'
```

## Notes
- `reports.view` and `reports.manage` are seeded by migration `006_reports.sql`
- PDF export is generated by a lightweight built-in writer to avoid adding new runtime dependencies
- report files are stored under `exports/reports/`
