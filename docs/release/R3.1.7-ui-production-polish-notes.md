# R3.1.7 UI production polish

สิ่งที่แก้รอบนี้
- ปรับ layout และ theme ทั้งระบบให้พร้อมใช้กับ browser จริง
- เพิ่มโลโก้และ branding ใน header/login
- เพิ่มกราฟเทรนด์ error/mismatch/incident/lag ในหน้า `/trends`
- เพิ่มหน้า report สำหรับ
  - `/reports/daily`
  - `/reports/admin`
  - `/reports/executive`
- ปรับหน้า `/reports` ให้เป็น report hub พร้อมภาพรวม 30 วัน
- ปรับหน้า `/access` และ `/access/audit` ให้อ่านง่ายและจัดการผู้ใช้สะดวกขึ้น
- เพิ่ม dashboard charts เพื่อใช้ทดสอบภาพรวมก่อนขึ้น production
- เพิ่ม static asset สำหรับ chart rendering (`highcharts.js`) และ logo (`moph.png`)
- อัปเดต smoke verification ให้รวม route ใหม่

ข้อควรรู้
- แพตช์นี้เน้น UI/UX และ browser workflow เป็นหลัก ไม่แตะ public endpoint เดิม
- route JSON เดิมของ `/trends` และ `/reports` ยังอยู่
- เนื่องจาก environment ในเครื่องทำงานนี้ไม่มี Flask จึงตรวจได้ระดับ syntax/Jinja parse เท่านั้น ต้องรัน verify/unittest บนเครื่องคุณอีกครั้ง
