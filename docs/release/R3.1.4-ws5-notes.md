# R3.1.4-ws5 notes

ก้อนนี้ปิดงาน WS5 ส่วนหลักตามเอกสาร Phase 3 โดยเน้น security/governance hardening แบบไม่รื้อ baseline เดิม

## สิ่งที่เพิ่ม
- session idle timeout และ absolute session lifetime สำหรับ browser session
- failed login counter + temporary lockout สำหรับผู้ใช้ที่เก็บใน DB
- password policy สำหรับ
  - เปลี่ยนรหัสผ่านเอง
  - สร้างผู้ใช้ใหม่
  - reset password โดย admin
- หน้า `/access/audit` สำหรับดู audit log ทั้งแบบ HTML และ JSON
- preview diff ของ config change request ก่อน approve/apply
- audit log เพิ่มตอน login fail และ report download

## migration
ใช้ไฟล์ `app/sql/007_security.sql`

## หมายเหตุ
- lockout ใช้กับ DB-backed users เท่านั้น ไม่ใช้กับ env admin
- migration เป็น additive ไม่ลบของเดิม
- หากเครื่องจริงยังไม่ได้ตั้ง env ใหม่ ระบบยังรันได้ด้วย default ที่ใส่ไว้ใน `app/config.py`

## ค่าคอนฟิกใหม่
- `SESSION_IDLE_MINUTES=30`
- `SESSION_ABSOLUTE_MINUTES=720`
- `LOGIN_LOCKOUT_THRESHOLD=5`
- `LOGIN_LOCKOUT_MINUTES=15`
- `PASSWORD_MIN_LENGTH=10`
- `PASSWORD_REQUIRE_UPPER=1`
- `PASSWORD_REQUIRE_LOWER=1`
- `PASSWORD_REQUIRE_DIGIT=1`
