R3.1.8 production UI fix

สิ่งที่แก้หลัก
- ปรับถ้อยคำหน้าเว็บจากโหมดทดสอบเป็นโหมดใช้งานจริง
- ทำให้ field input/select/textarea ใช้ class CSS ตามชนิดข้อมูลแบบเดียวกัน
- เพิ่ม JS ช่วยผูก class ให้ form control ที่ยังไม่ได้ระบุ class ตรงใน template
- ปรับ layout กราฟ trends/reports ให้แสดง 2 คอลัมน์บนจอใหญ่
- จำกัดการแสดงผลตารางใหญ่ใน trends และ daily report เป็น 20 แถวต่อรอบ
- ทำให้หน้า trends/jobs/alerts/reports ทนต่อกรณี table ใหม่ยังไม่ถูก migrate โดยไม่ล้ม 500 ทั้งหน้า
- เก็บแพตช์ให้เป็น runtime-only และไม่พ่วงไฟล์ test เข้ามาในชุดแพตช์
