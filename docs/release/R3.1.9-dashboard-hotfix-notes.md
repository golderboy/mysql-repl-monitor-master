# R3.1.9 dashboard hotfix

## สิ่งที่แก้
- แก้หน้าแรก (`/`) ไม่ให้ล้ม 500 จากตัวแปรกราฟ dashboard ที่ template เรียกใช้แต่ route ไม่ได้ส่งค่าไป
- เพิ่ม fallback หน้า dashboard เมื่อ table สรุปรายวัน/รายชั่วโมง หรือ table monitor บางส่วนยังไม่ถูก migrate ครบ
- แสดงข้อความเตือนบนหน้า dashboard แทนการล้มทั้งหน้า ถ้าข้อมูลบางชุดยังไม่พร้อม
- ปรับข้อความบน quick actions ให้เป็น wording สำหรับใช้งานจริง

## สาเหตุ
- template `dashboard.html` มี `dashboard-error-chart` และ `dashboard-lag-chart`
- route `dashboard()` ไม่ได้ส่ง `dashboard_error_chart` และ `dashboard_lag_chart` ทำให้ Jinja `tojson` พังและขึ้น 500
- ในบางเครื่อง production ยังอาจไม่มี `monitor_daily_stats` หรือ `monitor_hourly_lag_stats` ทำให้ query พังได้อีก

## ไฟล์ที่แก้
- `app/webapp.py`
- `app/templates/dashboard.html`
