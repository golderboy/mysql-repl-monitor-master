WS6 backup/restore drill

หลักการสำคัญ
- production runtime script ห้ามใช้ root
- production runtime script ห้าม import backup กลับเข้า MariaDB โดยใช้ app users
- การ restore database import เป็น DBA-controlled procedure แยก ทำบน non-production target เท่านั้น

สิ่งที่ runtime-safe path ต้องได้
- สำรอง db_monitor ได้ด้วย mon_backup
- สำรอง exports ได้
- มี SHA256SUMS
- ตรวจ bundle ได้โดยไม่ต้องใช้ privileged DB user

คำสั่ง runtime-safe
1) สร้าง backup
   bash scripts/backup_monitor.sh

2) ตรวจ bundle และเตรียมชื่อฐานทดสอบ
   bash scripts/drill_restore_test.sh /opt/mysql-repl-monitor/backups/<timestamp> db_monitor_restore_test

DBA-controlled restore procedure (นอก runtime users)
1) DBA เตรียม non-production MariaDB target
2) DBA สร้างฐาน db_monitor_restore_test
3) DBA import ไฟล์ SQL จาก backup ลงฐานทดสอบ
4) DBA ตรวจจำนวนตารางและ sample rows เช่น monitor_servers, monitor_settings, monitor_incidents
5) DBA บันทึกผล drill ลงเอกสาร/ตั๋วงาน
6) DBA ลบฐานทดสอบเมื่อจบ

ข้อห้าม
- ห้าม restore ทับ db_monitor live โดยไม่มี backup ล่าสุด
- ห้ามใช้ mon_app / mon_web_admin / mon_backup เพื่อ import database กลับเข้า DB
- ห้ามฝัง root password หรือ privileged DB credential ใน production script
