WS6 rollback

WS6 เป็นก้อนเอกสาร + script + backup timer เป็นหลัก จึง rollback ง่าย

1) หยุดและ disable timer backup
   systemctl disable --now mysql-repl-monitor-backup.timer
   rm -f /etc/systemd/system/mysql-repl-monitor-backup.timer
   rm -f /etc/systemd/system/mysql-repl-monitor-backup.service
   systemctl daemon-reload

2) ถ้าต้องการลบไฟล์ ws6 ออกจาก project
   rm -f scripts/preflight_check.sh
   rm -f scripts/postflight_check.sh
   rm -f scripts/backup_monitor.sh
   rm -f scripts/restore_monitor.sh
   rm -f scripts/drill_restore_test.sh
   rm -f scripts/config_diff.sh
   rm -f scripts/nightly_backup_cleanup.sh
   rm -f scripts/enable_backup_timer.sh
   rm -f deploy/mysql-repl-monitor-backup.service
   rm -f deploy/mysql-repl-monitor-backup.timer
   rm -f docs/WS6_RUNBOOK.txt
   rm -f docs/WS6_BACKUP_RESTORE_DRILL.txt
   rm -f docs/WS6_OPERATIONS_HANDOFF.txt
   rm -f docs/WS6_ROLLBACK.txt

3) ws6 ไม่ควรทำให้ phase 1-WS5 หยุด เพราะไม่แตะ runtime core
